# Database Operations Guide

> Common operations for database migrations and database files.

---

## 1. Current Database File Location

The database-related content of the current project is placed under the `backend/` directory:

```text
backend/migrations/
backend/instance/perth_explorer.db
```

- `backend/migrations/`: database migration directory
- `backend/instance/`: local instance directory used by Flask
- `backend/instance/perth_explorer.db`: local SQLite database file

---

## 2. Database Generation Process

The database file is generated through Flask-Migrate, not created directly. The generation process is:

1. Activate the virtual environment
2. Initialize the migration directory
3. Generate a migration script based on the models
4. Apply the migration and create database tables
5. Check whether the process succeeded
6. Continue using the migration tool to manage database structure changes during later development
7. Generate database tables in the target environment through the migration tool during deployment

---

## 3. Initialize Database Migrations and Database File

### 3.1 Activate the Virtual Environment

```bash
source .venv/bin/activate
```

---

### 3.2 Initialize the Migration Directory

```bash
flask --app backend.app db init
```

- `flask`: runs the Flask command-line tool
- `--app backend.app`: tells Flask that the application entry point is `backend.app`
- `backend.app`: means importing the Flask application object from `backend/app.py`
- `db`: calls the database migration command group provided by Flask-Migrate
- `init`: initializes the migration environment
- `flask --app backend.app db init`: creates the migration directory at the location specified by the project configuration

It will create:

```text
backend/migrations/
```

---

### 3.3 Generate a Migration Script Based on the Models

```bash
flask --app backend.app db migrate -m "initial schema"
```

- `migrate`: generates a migration script based on the differences between the current `backend/models.py` and the database state
- `-m`: adds a description message for this migration
- `"initial schema"`: the migration message, meaning this is the initial database structure
- `flask --app backend.app db migrate -m "initial schema"`: reads the current ORM models and generates a migration file

---

### 3.4 Apply the Migration and Create Database Tables

```bash
flask --app backend.app db upgrade
```

- `upgrade`: applies migration scripts that have not yet been applied to the actual database
- `flask --app backend.app db upgrade`: creates or updates database tables according to the migration scripts

It will create or update:

```text
backend/instance/perth_explorer.db
```

---

## 4. Check Whether It Succeeded

### 4.1 Check the Current Database Migration Version

```bash
flask --app backend.app db current
```

- `current`: checks which migration version has already been applied to the current database
- `flask --app backend.app db current`: checks which migration revision the database is currently on

Output:

```text
113a1b932956 (head)
```

- `113a1b932956` is the migration version ID generated by Flask-Migrate
- `(head)` means the current database is already at the latest migration version

---

### 4.2 Check Whether the Models and Migrations Are Consistent

```bash
flask --app backend.app db check
```

- `check`: checks whether the current models still have changes that have not been generated as migrations
- `flask --app backend.app db check`: compares the models and migration state to confirm whether there are any unhandled database structure changes

---

### 4.3 View the Actual Tables in SQLite

```bash
sqlite3 backend/instance/perth_explorer.db ".tables"
```

- `sqlite3`: SQLite command-line tool
- `backend/instance/perth_explorer.db`: database file
- `".tables"`: lists all tables in the current database
- `sqlite3 backend/instance/perth_explorer.db ".tables"`: directly checks which tables have been created in the actual database file

---

## Generate the Database During Deployment

Run this in the deployment environment:

```bash
flask --app backend.app db upgrade
```

This creates or updates database tables according to the migrations. The local SQLite database file will be generated at `backend/instance/perth_explorer.db` when using the default project configuration.
